# v2.4 - Atualizações de Segurança e Interface

## Resumo
Esta versão foca em melhorias significativas de segurança no backend, validações de dados e uma reformulação da responsividade para dispositivos móveis nas tabelas de gerenciamento.

## Alterações

### Backend (Segurança e Validação)
- **Correção de Login (Erro 400)**: Resolvido problema de chave duplicada ao atualizar tokens de refresh.
- **Cookies Seguros**: Implementação de `ResponseCookie` para tokens de acesso e refresh, garantindo `HttpOnly` e `SameSite=Strict`.
- **Validações DTO**: Adicionadas anotações de validação (`@NotBlank`, `@Size`, `@Email`) em:
  - `LoginRequestDTO`
  - `UpdateUserRequestDTO`
  - `UpdateRoleRequestDTO`
- **Novo DTO**: Criação de `ToolRequestDTO` para validação na criação e edição de ferramentas.

### Frontend (Interface e UX)
- **Responsividade Mobile**:
  - Implementação de visualização em cartões (Cards) para a tabela de **Usuários**.
  - Implementação de visualização em cartões para a lista de **Ferramentas Administrativas** (`/dashboard/tools/settings`).
  - Ocultação automática das tabelas em telas menores que `lg`.
- **Correções de UX**:
  - Remoção da mensagem "Sessão Expirada" ao carregar a página de login (`/auth`).
  - Correção do link "Configurações" no menu do usuário para apontar corretamente para `/dashboard/settings`.
  - Correção do input de filtro de usuários que perdia o foco ao digitar.

### Correções Gerais
- **Clipboard**: Implementação de utilitário seguro para cópia (clipboard), tratando erros de permissão.
- **Geração de UUID**: Correção no gerador de dados para evitar erros de `crypto.randomUUID` em certos contextos.
